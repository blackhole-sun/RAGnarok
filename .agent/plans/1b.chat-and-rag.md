# 1b. Chat + RAG: OpenAI Responses API + UI

## Complexity: ‚öÝÔ∏è Medium

## Context
Builds on 1a (auth working, schema in place). Implements the full chat interface (threaded conversations with SSE streaming), file ingestion to OpenAI's vector store, and LangSmith tracing. Uses the OpenAI Responses API with `previous_response_id` for thread chaining and `file_search` for RAG.

## Pre-Requisites
- 1a completed and validated
- `backend/.env` and `frontend/.env.local` populated with real values

## Tasks

### Task 1b-1: Thread Backend (Models + CRUD Routes)
- Create `backend/app/models/__init__.py`
- Create `backend/app/models/thread.py` ‚Äî Pydantic: `Thread`, `CreateThreadRequest`, `UpdateThreadRequest`
- Create `backend/app/routers/threads.py`:
  - `GET /api/threads` ‚Äî list user's threads ordered by `updated_at DESC`
  - `POST /api/threads` ‚Äî create thread, return full Thread object
  - `PATCH /api/threads/{thread_id}` ‚Äî update title (verify ownership)
  - `DELETE /api/threads/{thread_id}` ‚Äî delete thread (CASCADE handles messages)
- Wire into `main.py`
- **Validation:** `curl` all 4 endpoints with valid Bearer token; RLS verified (different user gets empty list)

### Task 1b-2: Message Backend
- Create `backend/app/models/message.py` ‚Äî Pydantic: `Message`
- Create `backend/app/routers/messages.py`:
  - `GET /api/messages/{thread_id}` ‚Äî verify user owns thread, return messages ASC by `created_at`
- Wire into `main.py`
- **Validation:** Insert test messages directly in Supabase SQL editor, call endpoint, confirm ordered correctly

### Task 1b-3: OpenAI Service + LangSmith
- Create `backend/app/services/__init__.py`
- Create `backend/app/services/openai_service.py`:
  - Sync `stream_chat_response(user_message, previous_response_id, vector_store_id)` generator
  - Decorated with `@traceable(name="chat_with_responses_api", run_type="llm")`
  - Yields `("delta", str)` for each text delta
  - Yields `("done", response_id_str)` when complete
  - Yields `("error", error_msg)` on exception
- **Validation:** Call from Python REPL with test message; confirm events print; LangSmith dashboard shows trace

### Task 1b-4: Chat Stream Endpoint
- Create `backend/app/models/chat.py` ‚Äî Pydantic: `ChatRequest` (thread_id, message)
- Create `backend/app/routers/chat.py`:
  - `POST /api/chat/stream` returns `StreamingResponse` with `media_type="text/event-stream"`
  - Flow: verify thread ownership ‚Üí get `last_response_id` + `vector_store_id` from thread (or env) ‚Üí call `stream_chat_response` ‚Üí stream SSE deltas ‚Üí on done: save user+assistant messages to DB, update `threads.last_response_id`
- SSE format: `event: <type>\ndata: <json>\n\n`
- Headers: `Cache-Control: no-cache`, `X-Accel-Buffering: no`, `Connection: keep-alive`
- Wire into `main.py`
- **Validation:** `curl -N -X POST -H "Authorization: Bearer <jwt>" -H "Content-Type: application/json" -d '{"thread_id":"...","message":"hello"}' http://localhost:8000/api/chat/stream` ‚Üí delta events stream to terminal

### Task 1b-8: Ingestion Backend
- Create `backend/app/models/ingestion.py` ‚Äî Pydantic: `UploadedFile`, `UploadResponse`
- Create `backend/app/services/ingestion_service.py`:
  - `upload_file_to_openai(file_bytes, filename, mime_type, user_id)` ‚Äî async def using `BackgroundTasks`
  - Step 1: Insert `uploaded_files` row with `status='pending'`
  - Step 2: `client.files.create(file=(filename, file_bytes, mime_type), purpose="assistants")`
  - Step 3: Get or create vector store ‚Äî check env var `OPENAI_VECTOR_STORE_ID`; if empty, `client.vector_stores.create(name="rangarok-rag")` and print the ID to set in env
  - Step 4: `client.vector_stores.files.create(vector_store_id=..., file_id=...)` ‚Üí update status to `'processing'`
  - Background poll: every 5s call `client.vector_stores.files.retrieve()`, update DB status until `completed` or `failed`
  - `delete_file(file_id, user_id)` ‚Äî delete from OpenAI + DB
  - `list_files(user_id)` ‚Äî return all user's uploaded_files
- Create `backend/app/routers/ingestion.py`:
  - `POST /api/ingestion/upload` ‚Äî accepts `multipart/form-data`, calls service
  - `GET /api/ingestion/files` ‚Äî list files
  - `DELETE /api/ingestion/files/{file_id}` ‚Äî delete
- Wire into `main.py`; add `OPENAI_VECTOR_STORE_ID` to config
- **Validation:** `curl -F "file=@test.pdf" -H "Authorization: Bearer <jwt>" http://localhost:8000/api/ingestion/upload` ‚Üí DB record created; after ~30s status = `completed`

