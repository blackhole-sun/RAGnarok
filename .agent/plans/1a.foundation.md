# 1a. Foundation: Project Setup + Auth

## Complexity: ⚠️ Medium

## Context
Establishes the project skeleton: Python/FastAPI backend with venv, React/Vite frontend with shadcn/ui, Supabase schema with RLS on all 4 tables, email/password auth, and environment variable wiring. This is the prerequisite for all Module 1 chat/RAG work.

## Pre-Requisites (Manual — Developer Provides)
1. Supabase project created → `SUPABASE_URL`, `SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`, `SUPABASE_JWT_SECRET`
2. OpenAI API key → `OPENAI_API_KEY`
3. LangSmith account + project → `LANGSMITH_API_KEY`, project name

## Tasks

### Task 1a-1: Backend Scaffold
- Create `backend/` directory
- Run `python3 -m venv venv` inside `backend/`
- Create `backend/requirements.txt`
- Create `backend/main.py` — FastAPI app, CORS, router includes, `/api/health`
- Create `backend/app/__init__.py`
- Create `backend/app/config.py` — pydantic-settings BaseSettings reads all env vars
- **Validation:** `cd backend && source venv/bin/activate && pip install -r requirements.txt && uvicorn main:app --reload` → `GET http://localhost:8000/api/health` returns `{"status": "ok"}`

### Task 1a-2: Supabase Schema
- Create `supabase/migrations/001_init.sql` with all DDL
- Run in Supabase SQL editor
- **Validation:** 4 tables visible in Table Editor; RLS enabled on all; sign up test user → profile row auto-created

### Task 1a-3: Env Files
- Create `backend/.env` (from template, developer fills values)
- Create `frontend/.env.local` (from template, developer fills values)
- Create root `.env.example` documenting all vars
- Update `.gitignore` to exclude `backend/.env`, `frontend/.env.local`, `backend/venv/`, `node_modules/`
- **Validation:** `python -c "from app.config import settings; print(settings.OPENAI_API_KEY)"` prints key

### Task 1a-4: JWT Auth Dependency
- Create `backend/app/database.py` — Supabase client using service role key
- Create `backend/app/dependencies.py` — `get_current_user` JWT verification with PyJWT
- Create `backend/app/routers/__init__.py`
- Create `backend/app/routers/auth.py` — `GET /api/auth/me`
- Wire router into `main.py`
- **Validation:** Valid JWT → 200 with user data; no token or expired → 401

### Task 1a-5: Frontend Scaffold
- `npm create vite@latest frontend -- --template react-ts` from project root
- `cd frontend && npm install`
- Install: `npm install @supabase/supabase-js @tanstack/react-query zustand react-router-dom`
- Init Tailwind: `npm install -D tailwindcss postcss autoprefixer && npx tailwindcss init -p`
- Init shadcn: `npx shadcn@latest init` (default style, Zinc, CSS variables)
- Add shadcn components: `npx shadcn@latest add button input card scroll-area separator badge textarea`
- Create `frontend/src/lib/utils.ts` if not auto-generated
- **Validation:** `npm run dev` → `http://localhost:5173` loads, no console errors

### Task 1a-6: Auth Store + Supabase Client
- Create `frontend/src/lib/supabase.ts` — Supabase client using VITE_ env vars
- Create `frontend/src/store/authStore.ts` — Zustand store: `{ user, session, setUser, setSession }`
- Create `frontend/src/hooks/useAuth.ts` — wraps signInWithPassword, signUp, signOut, onAuthStateChange
- Create `frontend/src/types/index.ts` — TypeScript interfaces for Thread, Message, User
- **Validation:** `supabase.auth.getSession()` called from a test component returns null on fresh load

### Task 1a-7: Auth UI + Routing
- Create `frontend/src/components/auth/AuthForm.tsx` — single form toggling between Login/Signup
- Create `frontend/src/components/auth/AuthGuard.tsx` — redirects to `/login` if no session
- Create `frontend/src/pages/LoginPage.tsx`
- Create `frontend/src/pages/ChatPage.tsx` — placeholder "Chat coming soon"
- Create `frontend/src/pages/IngestionPage.tsx` — placeholder "Ingestion coming soon"
- Wire `frontend/src/App.tsx` with React Router: `/login`, `/` (guarded → ChatPage), `/ingest` (guarded → IngestionPage)
- Wrap app with `QueryClientProvider` and `BrowserRouter` in `main.tsx`
- **Validation:** Navigate to `/` → redirect to `/login`; sign up → redirect to `/`; refresh → stays logged in
